## 1. Latency (full ASGI request path, instant mocked upstream)

Hit latencies are end-to-end through Arca. Miss latency here is Arca's
added overhead only (canonicalize + embed + local L2 search + tee); a real
miss additionally pays the normal Anthropic round trip.

## 2. Semantic threshold quality (18-pair eval set)

Cosine similarity between each pair, embedded with BAAI/bge-small-en-v1.5.
`hit` pairs are paraphrases that should match; `miss` pairs are near-miss
lookalikes (including negation) that must not.

| Threshold | Paraphrase recall | False positives |
|---|---|---|
| 0.90 | 9/9 | 3/9 |
| 0.92 | 7/9 | 1/9 |
| 0.95 | 4/9 | 0/9 |

End-to-end through the proxy at the default threshold (0.95): 4/9 paraphrase hits, 0/9 false hits on negative controls.

## 3. Estimated cost avoided (model-based, stated assumptions)

Assumptions: every request is 1500 input / 350 output tokens on claude-sonnet-4-6 (3.00 / 15.00 USD per MTok). These are assumptions, not measurements.

- Workload: 107 requests, 44 served from cache (41% hit rate in this synthetic workload)
- Cost per uncached call: $0.009750
- API spend avoided: $0.4290 vs $1.0433 without caching (41%)

The hit rate above is a property of this synthetic workload (every cold
prompt repeated once, plus paraphrase pairs), not a claim about real Claude
Code traffic. Real multi-turn agent sessions repeat far less; see the
Limitations section of the README.
